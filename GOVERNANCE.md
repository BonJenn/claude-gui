# Governance

Blackcrab is currently maintained by Jonathan Benn.

## Decision Making

The maintainer makes final decisions on scope, design, releases, and project
direction. Community feedback is welcome, but the roadmap is advisory until
work is accepted and merged.

## Contribution Review

Pull requests are reviewed for:

- User impact.
- Stability risk.
- Security implications.
- Fit with the existing architecture.
- Test and manual-QA coverage.
- Documentation updates when behavior changes.

Large changes should start as an issue or discussion before implementation.

## Releases

Releases are cut by the maintainer using `RELEASE.md`.

## Conduct

All project spaces follow `CODE_OF_CONDUCT.md`.
